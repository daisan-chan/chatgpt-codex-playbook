# AGENTS.md

## このリポジトリでの Codex 編集方針
- このリポジトリはドキュメント運用専用です。実装コード・スクリプト・CI 設定は追加しません。
- 変更は常に小さく、レビューしやすい単位に分割します。
- 指示にない運用ルールや手順を新規に発明せず、既存ドキュメントとの整合を優先します。
- `README.md`、`docs/00_project_playbook.md`、本ファイルの記述に矛盾を作らないでください。

## 変更前の必須読了順
1. `README.md`
2. `docs/00_project_playbook.md`
3. `docs/02_threading_and_branching.md`
4. `docs/03_codex_prompt_template.md`
5. `docs/04_review_checklist.md`

## ドキュメント構成ルール
- ファイル名は既存命名を維持します。
- `docs/` 配下の番号順（00, 01, 02...）を維持します。
- 既存セクションを削除する場合は、目的と影響を変更内容に明記します。
- 追記時は「どの運用判断に使う情報か」が分かる見出しを付けます。

## Codex 完了報告フォーマット
最終応答は以下の順で簡潔に報告してください。
1. 変更ファイル一覧
2. 各ファイルの要約（1〜2 行）
3. 実施した確認コマンド（該当がない場合は `なし` と記載可）
4. 未解決事項・前提（あれば）
5. 次の小さな推奨ステップ
