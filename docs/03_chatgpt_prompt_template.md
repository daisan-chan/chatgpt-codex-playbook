# 03 ChatGPT プロンプトテンプレート

## 目的
このテンプレートは、まだ曖昧な構想や要望を `docs/02_project_brief_template.md` に沿ったプロジェクト概要へ整理するための、ChatGPT 用プロンプトです。プロジェクト開始時に前提をそろえ、幹スレッドで扱う判断材料を整えやすくすることを目的にしています。Codex 実行前の整理段階で使うことを想定しています。

このプロンプトは、`docs/02_project_brief_template.md` と関連サンプルが ChatGPT プロジェクトの情報源に登録されている前提で使います。この前提がない場所で使う場合は、テンプレ本文または対象項目一覧も一緒に渡してください。

## このテンプレートを使う場面
- プロジェクト開始時に、要望がまだ抽象的で整理しきれていないとき
- `docs/02_project_brief_template.md` を埋める前に、項目の過不足を確認したいとき
- 幹スレッドへ持ち込む判断材料を、短時間で構造化したいとき

## ChatGPT に期待すること
- 曖昧な要望を整理する
- `目的` と `背景` を分ける
- `対象範囲` と `対象外` を分ける
- 情報不足や未確定事項を明示する
- 完了条件を確認可能な表現にする
- 後続の幹スレッド、枝スレッド、Codex 実行につながる粒度にする

## ChatGPT にやってほしくないこと
- 勝手に実装詳細や技術設計へ広げること
- 未確定事項を断定すること
- 対象範囲を広げすぎること
- いきなり Codex 向け実行指示まで飛ぶこと
- 人が判断すべきことを ChatGPT が確定事項として決めてしまうこと

## 出力形式
1. 整理した前提
2. 不明点 / 確認したい点
3. `docs/02_project_brief_template.md` に沿った記入案
4. 対象外にしたほうがよい候補
5. 次に人が判断すべきこと

## コピペ用プロンプト
```md
以下の構想を、`docs/02_project_brief_template.md` に沿って整理してください。

前提:
- `docs/02_project_brief_template.md` と関連サンプルが ChatGPT プロジェクトの情報源に登録されている前提で整理してください。
- もし情報源がない場合は、提示した本文と項目一覧を前提に整理してください。

指示:
- 情報不足があれば、勝手に断定せず `不明点` または `仮置きの前提` として明示してください。
- 実装詳細や技術設計には広げず、プロジェクト概要の整理に集中してください。
- `目的` と `背景` を分けてください。
- `対象範囲` と `対象外` を分けてください。
- `完了条件` は確認可能な表現にしてください。
- `ChatGPT / Codex 引き継ぎメモ` まで含めてください。
- 最後に `次に人が判断すべきこと` を示してください。

出力形式:
1. 整理した前提
2. 不明点 / 確認したい点
3. プロジェクト概要の記入案
   - プロジェクト名
   - 目的
   - 背景
   - 対象範囲
   - 対象外
   - 関係者
   - 成果物
   - 制約
   - 完了条件
   - ChatGPT / Codex 引き継ぎメモ
4. 対象外にしたほうがよい候補
5. 次に人が判断すべきこと

構想メモ:
- （ここに箇条書きメモや要望を書く）
```

## 使い方メモ
- 最初から全項目を埋めていなくても使えます。
- 箇条書きメモや雑な構想から始めても問題ありません。
- 1 回で完成させず、何回か往復して精度を上げる前提で使ってください。
- 出力結果はそのまま採用せず、人が最終判断して `docs/02_project_brief_template.md` へ反映してください。
