# 01 運用モデル図

この図は `docs/00_project_playbook.md` の補助図です。作業の場（ChatGPT プロジェクト / Codex / GitHub）と、運用の主フロー（幹スレッド → 枝スレッド → Codex 作業 → Gitブランチ → プルリク）に加えて、マージ後の資産反映と、幹スレッド・枝スレッドが参照する共有情報源の関係を分けて示します。

```mermaid
flowchart TB
    H[人]

    subgraph S1[ChatGPT プロジェクト]
        M[幹スレッド]
        B[枝スレッド]
        A[プロジェクト情報源]
    end

    subgraph S2[Codex]
        X[Codex 作業]
    end

    subgraph S3[GitHub]
        G[Gitブランチ]
        P[プルリク]
    end

    M -->|方針を渡す| B
    H -->|Codex へ実行指示| X
    X -->|編集・差分提示| G
    G --> P
    P -->|レビュー指摘・修正方針| B
    P -->|マージ済み資産を反映| A
    A -.->|最新資産を参照| M
    A -.->|最新資産を参照| B
    B -->|採用内容・次ステップを幹へ報告| M

    H -->|方針決定| M
    H -->|必要に応じた判断・確認| B
    H -->|レビュー・承認| P
```

- ChatGPT プロジェクト / Codex / GitHub の3つを、作業の場として分けて配置しています。
- 幹スレッドと枝スレッドは ChatGPT プロジェクト上の場として扱い、主フローは上から下へ追える構成にしています。
- 人は外側に置き、方針決定・判断確認・レビュー承認の主体として関与します。
- レビュー差し戻し時は枝スレッドで修正方針を再整理し、同じ Gitブランチを更新します。
- マージ後は ChatGPT 側のプロジェクト情報源を更新し、幹スレッドと枝スレッドの両方が最新資産を参照します。
- 枝スレッドは実行内容と修正方針を整理する場で、Codex への実行指示は人が行います。
- 幹スレッドへの確定報告は、枝スレッドで整理した採用内容と次ステップを戻す形で扱います。
