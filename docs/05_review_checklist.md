# 05 レビューチェックリスト

このチェックリストは、Codex 生成物の自己点検と人のレビューの両方で使うための最終確認表です。マージ前に、構成・整合・スコープの抜け漏れがないかを短時間で確認することを目的にしています。

## 構造
- [ ] 変更対象は指示されたファイルに限定されている
- [ ] 見出し構造が読みやすく、目的が明確
- [ ] README と docs の参照関係が成立している
- [ ] ChatGPT Project の情報源として読んだときの参照順が分かる

## 3 層構造の整合
- [ ] 上位幹 / 中間幹 / 枝の役割差が明記されている
- [ ] 3 層の区分が情報量ではなく判断の種類で説明されている
- [ ] 中間幹を作る条件が明記されている
- [ ] 枝化条件が明記されている
- [ ] 戻し先と戻し方の最小セットがある

## 自己認識と介入
- [ ] 各スレッドの自己認識項目が定義されている
- [ ] 自己認識テンプレートまたは導線がある
- [ ] 誤投入補正の導線がある
- [ ] スレッド移行テンプレートの導線がある
- [ ] 命名提案または命名変更提案のルールがある

## 命名と整合
- [ ] ファイル名・番号順・見出し名が既存ルールと一致
- [ ] 人 / ChatGPT / Codex の役割記述に矛盾がない
- [ ] 用語（上位幹 / 中間幹 / 枝 / プルリク）が統一されている
- [ ] Project 指示文テンプレートと各 docs の説明が矛盾していない

## スコープ管理
- [ ] 要件外の新規ファイルや仕組みを追加していない
- [ ] 実装詳細ではなく運用ガイドに集中している
- [ ] 変更が小さく、レビュー可能な粒度である
- [ ] 主対象外ファイルに不要な変更を広げていない

## テンプレート有用性
- [ ] テンプレートがそのまま再利用できる
- [ ] 入力すべき項目が不足なく、過剰でもない
- [ ] 記入例が短く実用的
- [ ] `docs/06_thread_operation_templates.md` への導線がある

## マージ準備
- [ ] プルリク説明に目的・差分・確認結果が記載されている
- [ ] 未解決事項とフォローアップが明示されている
- [ ] 上位幹または中間幹へ返す報告内容が整理されている
- [ ] 必要に応じて、マージ後に ChatGPT Project 情報源へ反映する手順が整理されている
