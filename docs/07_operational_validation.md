# 07 実運用検証メモ

この文書は、playbook を実運用しながら検証し、どの運用が効いたか、どこでズレたか、何を改善候補として戻すかを記録するための受け皿です。理想論を増やす場所ではなく、次の差分更新につなぐ観察メモとして使います。

## 何を検証する文書か
- 上位幹 / 中間幹 / 枝の 3 層分けが実運用で機能しているか
- 各スレッドの自己認識が、誤投入や階層逸脱の補正に役立っているか
- Project 指示文の参照順が、ChatGPT の振る舞いを安定させているか
- 移行テンプレートや命名提案が、実務の手戻りを減らしているか

## 観察ポイント
運用後に、最低限次を観察します。

- 上位幹に詳細作業が流れ込みやすくないか
- 中間幹が必要なテーマを、上位幹で抱え込み続けていないか
- 枝が全体判断まで抱え込んでいないか
- ChatGPT が自己認識項目を確認できているか
- 誤投入補正や枝化提案が自然に行えているか
- 命名が実態とずれたとき、提案で補正できているか
- Project 指示文が長すぎて運用しづらくなっていないか

## 問題記録欄
必要に応じて、以下を追記して記録します。

```md
### 記録日
- YYYY-MM-DD

### 対象プロジェクト / スレッド
- 

### 起きたこと
- 

### どの階層でズレたか
- 上位幹 / 中間幹 / 枝 / Project 指示文 / テンプレート

### 影響
- 

### 暫定対応
- 
```

## 改善候補欄
問題を見つけたら、すぐ本編へ書き換える前に、改善候補としてここへ仮置きします。

```md
### 改善候補
- 背景:
- 直したいズレ:
- 反映候補ファイル:
- 追記で足りるか / 既存記述の見直しが要るか:
- 反映前に追加で観察したいこと:
```

## 暫定運用メモ欄
正式反映の前に、チーム内で一時的に試すルールや言い回しがあればここへ記録します。

```md
### 暫定運用メモ
- 試したいこと:
- 適用期間:
- 適用対象:
- 期待する効果:
- 見直し予定日:
```

## 更新ルール
- 問題を見つけたら、まずこの文書へ観察事実として残す
- すぐ本編へ広げず、複数回の運用で再現するかを確認する
- 反映先は役割に応じて分ける
  - 原則の更新: `docs/00_project_playbook.md`
  - 実務ルールの更新: `docs/01_threading_and_branching.md`
  - ChatGPT 指示文の更新: `docs/03_chatgpt_prompt_template.md`
  - Codex 依頼の更新: `docs/04_codex_prompt_template.md`
  - テンプレートの更新: `docs/06_thread_operation_templates.md`
- README に影響する変更は、README も同時に見直す
- 大きな思想変更ではなく、既存運用を支える差分改善を優先する
