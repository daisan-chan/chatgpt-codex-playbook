# 00 プロジェクト運用プレイブック（運用ガイド）

## 目的
ChatGPT と Codex を使ったドキュメント中心の開発運用を、再現可能な手順として揃えることを目的とします。

## 対象範囲
- 作業の進め方
- スレッド運用とブランチ運用の対応
- 指示テンプレートとレビュー手順

## 対象外
- プロダクト実装の設計詳細
- 技術スタック固有のベストプラクティス集
- 自動化基盤（CI/CD）構築

## 運用モデル
- **幹スレッド**: 方針決定、優先順位、最終合意を扱う
- **枝スレッド**: 個別タスクの具体化と実行指示を扱う
- **Gitブランチ / プルリク**: 変更の実体を管理し、レビュー可能にする

## 責任分担
- **人**: 目的設定、意思決定、承認
- **ChatGPT**: 論点整理、文書ドラフト、レビュー支援
- **Codex**: 指示に基づく編集、差分提示、作業報告

## 標準ライフサイクル
1. **開始**: 幹スレッドで目的・制約・完了条件を定義
2. **枝スレッドで議論**: 枝スレッドで対象変更を小さく分割
3. **Codex 実行**: Codex が指示どおりに編集・確認
4. **レビュー**: チェックリストで整合性と過不足を確認
5. **プルリク**: GitHub で差分をレビューし、必要修正を反映
6. **幹スレッドへ報告**: 幹スレッドに結果・判断事項・次ステップを戻す

## 保守原則
- 小さく安全な変更を優先する
- 既存ルールとの整合を優先し、重複記述を避ける
- テンプレートは実運用で使える最小構成を維持する
- 方針変更時は README と関連ドキュメントを同時更新する
