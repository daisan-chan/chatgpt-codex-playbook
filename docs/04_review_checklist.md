# 04 Review Checklist

## Structure（構造）
- [ ] 変更対象は指示されたファイルに限定されている
- [ ] 見出し構造が読みやすく、目的が明確
- [ ] README と docs の参照関係が成立している

## Naming & Consistency（命名と整合）
- [ ] ファイル名・番号順・見出し名が既存ルールと一致
- [ ] Human / ChatGPT / Codex の役割記述に矛盾がない
- [ ] 用語（main thread / branch thread / PR）が統一されている

## Scope Control（スコープ管理）
- [ ] 要件外の新規ファイルや仕組みを追加していない
- [ ] 実装詳細ではなく運用ガイドに集中している
- [ ] 変更が小さく、レビュー可能な粒度である

## Template Usefulness（テンプレート有用性）
- [ ] テンプレートがそのまま再利用できる
- [ ] 入力すべき項目が不足なく、過剰でもない
- [ ] 記入例が短く実用的

## Merge Readiness（マージ準備）
- [ ] PR 説明に目的・差分・確認結果が記載されている
- [ ] 未解決事項とフォローアップが明示されている
- [ ] main thread へ返す報告内容が整理されている
