# 06 スレッド運用テンプレート集

この文書は、上位幹 / 中間幹 / 枝を実運用する際に、そのままコピペして使える最小テンプレート集です。長文の理想論ではなく、開始・移行・戻し・命名提案をすぐ回せる粒度にそろえます。

## 使い方
- 新規スレッド開始時は、まず「各スレッド共通の自己認識テンプレート」を使う
- 迷ったら、親スレッド・戻し先・終着点が書けるかで階層を確認する
- 誤投入や階層逸脱が起きたら、補正テンプレートまたは提案テンプレートを使う
- 命名は固定ルールで縛らず、実態とずれたときに提案で補正する

```md
- 扱わない範囲:
- 戻し先:
- 終着点:

もし会話内容がこの階層に合わない場合は、適切な移行先を提案してください。
```

## 上位幹開始テンプレート
```md
このスレッドは上位幹として開始します。

- テーマ: プロジェクト全体方針
- 目的: 全体方針、優先順位、完了条件を管理する
- 扱う範囲: 横断判断、優先順位、制約、中間幹作成判断
- 扱わない範囲: 個別タスクの詳細化、試行錯誤、文面の詰め
- 戻し先: 人の最終判断
- 終着点: 次に扱うテーマや枝の方向が決まっていること

詳細作業が必要になった場合は、中間幹または枝への切り出しを提案してください。
```

## 中間幹開始テンプレート
```md
このスレッドは中間幹として開始します。

- 親スレッド:
- テーマ:
- 目的: このテーマを継続管理し、関連枝の戻し先にする
- 扱う範囲: 論点整理、保留事項、関連枝、次の枝候補
- 扱わない範囲: 全体優先順位の再設計、1 タスクの細部作業の継続
- 戻し先: 上位幹
- 終着点: このテーマで判断すべきことが整理され、必要な枝が回っていること

具体タスクの深掘りが必要になったら、枝化を提案してください。
```

## 枝開始テンプレート
```md
このスレッドは枝として開始します。

- 親スレッド:
- テーマ:
- 目的: 1 つの具体タスクを完了条件に沿って進める
- 扱う範囲: 具体化、詳細化、試行錯誤、Codex 指示、差分確認
- 扱わない範囲: 全体方針の再設計、複数テーマの継続管理
- 戻し先: 中間幹 / 上位幹
- 終着点: 結果と未解決事項を親へ戻せること

もし全体方針の再判断が必要なら、親スレッドへ戻す提案をしてください。
```
